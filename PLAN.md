# План реализации ChatList

Поэтапный порядок работ на основании [PROJECT.md](PROJECT.md).

---

## Этап 1. Подготовка проекта

1. Обновить структуру каталогов и зависимости:
   - `main.py` — точка входа и GUI
   - `db.py` — работа с SQLite
   - `models.py` — бизнес-логика моделей и промтов
   - `network.py` — HTTP-запросы к API нейросетей
   - `.env.example` — шаблон переменных окружения для API-ключей
2. Добавить в `requirements.txt`: `httpx` (или `requests`), `python-dotenv`.
3. Зафиксировать схему БД в [DATABASE.md](DATABASE.md).

---

## Этап 2. Модуль базы данных (`db.py`)

1. Реализовать подключение к SQLite (файл `chatlist.db` в каталоге приложения).
2. Создать таблицы при первом запуске: `prompts`, `models`, `results`, `settings`.
3. Добавить CRUD-функции для каждой таблицы:
   - промты: создание, выборка, поиск по тексту и тегам;
   - модели: создание, обновление, получение активных (`is_active = 1`);
   - результаты: пакетное сохранение выбранных строк;
   - настройки: чтение и запись пар ключ–значение.
4. Инкапсулировать все SQL-запросы внутри `db.py` — остальные модули не обращаются к SQLite напрямую.

---

## Этап 3. Модуль моделей (`models.py`)

1. Описать dataclass/Pydantic-структуры: `Prompt`, `Model`, `Result`, `TempResult`.
2. Реализовать загрузку активных моделей из БД.
3. Реализовать работу с промтами: новый ввод, выбор из сохранённых.
4. Реализовать временную таблицу результатов в памяти (`list[TempResult]`):
   - поля: `model_name`, `response`, `selected`;
   - методы: добавление, очистка, получение выбранных, смена флага `selected`.
5. Реализовать сохранение выбранных результатов в БД через `db.py`.

---

## Этап 4. Модуль сети (`network.py`)

1. Загружать API-ключи из `.env` по имени переменной (`api_id` из таблицы `models`).
2. Реализовать базовую функцию отправки промта в одну модель (POST к `api_url`).
3. Добавить обработку ошибок: таймаут, HTTP-ошибки, отсутствие ключа в `.env`.
4. Реализовать параллельную или последовательную отправку промта во все активные модели.
5. *(Опционально)* Поддержка разных типов API: OpenAI-совместимый формат, DeepSeek, Groq.

---

## Этап 5. Основной интерфейс (`main.py`)

1. Заменить заглушку на полноценное окно PyQt6.
2. **Блок ввода промта:**
   - многострочное поле текста;
   - выпадающий список сохранённых промтов;
   - кнопка «Отправить».
3. **Таблица результатов** (временная, в памяти):
   - колонки: модель, ответ, чекбокс «Выбрать»;
   - обновление после получения ответов от всех моделей.
4. **Кнопка «Сохранить»** — запись строк с `selected = True` в таблицу `results`, очистка временной таблицы.
5. **Обработка нового запроса** — при вводе/отправке нового промта временная таблица очищается до получения новых ответов.
6. Индикатор загрузки на время ожидания ответов от API.

---

## Этап 6. Управление моделями и настройками (GUI)

1. Окно или вкладка «Модели»:
   - список моделей с полями: имя, URL, имя переменной ключа, активна/неактивна;
   - добавление, редактирование, включение/отключение модели.
2. Окно «Настройки»:
   - параметры из таблицы `settings` (таймаут запросов, путь к БД и т.д.).
3. Окно «История»:
   - просмотр сохранённых промтов и результатов из БД.

---

## Этап 7. Дополнительные функции (опционально)

1. Поиск и сортировка во всех таблицах GUI.
2. Экспорт выбранных результатов в Markdown и JSON.
3. Логирование запросов в файл (`logs/chatlist.log`).
4. Сборка исполняемого файла через PyInstaller (`build.ps1`).

---

## Этап 8. Тестирование и финализация

1. Проверить сценарий: ввод промта → отправка → выбор строк → сохранение.
2. Проверить сценарий: выбор сохранённого промта → повторная отправка.
3. Проверить поведение при ошибках API и отсутствии ключей в `.env`.
4. Обновить `README.md`: установка, настройка `.env`, запуск, сборка `.exe`.

---

## Порядок зависимостей между этапами

```
Этап 1 → Этап 2 → Этап 3 → Этап 4 → Этап 5 → Этап 6 → Этап 7 → Этап 8
                ↘___________________↗
```

Этапы 2–4 можно частично вести параллельно после этапа 1, но GUI (этап 5) собирается только после готовности `db.py`, `models.py` и `network.py`.
